## 1. 快速入门

### 1.1 安装插件

- **在线安装（待发布至 VS Code Marketplace）**：在扩展商店搜索「HIT Smart Coder」并安装。
- **手动安装（当前方式）**：
  1. 下载插件 `.vsix` 文件（或克隆 GitHub 仓库）。
  2. 在 VS Code 中按 `Ctrl+Shift+P`，输入 `Extensions: Install from VSIX...`，选择文件。
  3. 重载窗口。

### 1.2 配置 API 密钥（云端模式必选）

插件默认使用通义千问云端 API，需获取 APIKey 并配置：

1. 打开 VS Code 设置 (`Ctrl+,`)，搜索 `hitsmartcoder.apiKey`。
2. 填入你的通义千问 APIKey。

### 1.3 第一次体验

- 打开任意代码文件（如 `test.py`）。
- 将光标放在一个函数内，按下 `Ctrl+Shift+P` 输入 `Smart Complete: 触发智能补全`，AI 将为你生成代码。
- 选中一行代码，右键点击 `AI: 解释这段代码`，查看中文解释。

## 2. 功能详解

### 2.1 右键菜单功能

| 菜单项 | 功能 | 使用方式 |
| --- | --- | --- |
| 快速插入 Console Log | 一键生成当前语言的调试打印语句 | 选中变量名 → 右键 → 选择菜单项 |
| AI: 解释这段代码 | 用中文解释选中代码的作用 | 选中代码块 → 右键 → 解释 |
| AI: 智能补全 | 基于上下文生成代码补全建议 | 光标停在需要补全处 → 右键 → 补全 |

**示例**：选中变量 x，点击「快速插入 Console Log」，自动生成 `console.log(x)`（JavaScript）或 `print(x)`（Python）。

### 2.2 智能代码补全（自动触发）实验功能

- 在 Python / JavaScript / C++ 等语言下，输入代码时插件将自动显示灰字建议（类似 Copilot）。
- 按 `Tab` 接受建议，`Esc` 忽略。
- 需手动开启：设置 `hitsmartcoder.inlineCompletion.enabled` 为 `true`。

### 2.3 多候选列表（手动触发）

- 通过命令面板 `Smart Complete: 显示多个候选`，会弹出 2~3 个不同实现方案，按 `Enter` 选择插入。
- 适合对比学习或选择最优实现。

## 3. 配置说明

所有配置项均在 VS Code settings.json 中设置（搜索前缀 `hitsmartcoder`）。

### 3.1 基础配置

| 配置项 | 描述 | 默认值 | 可选值 |
| --- | --- | --- | --- |
| apiKey | 通义千问 APIKey | `""` | 字符串 |
| completionMode | 补全模式 | `"standard"` | standard 直接补全<br/>hint 教学提示<br/>scaffold 框架填空<br/>explain 讲解引导 |
| modelBackend | 推理后端 | `"cloud"` | cloud 云端（默认）<br/>local 本地微调模型（需自建服务） |
| localEndpoint | 本地模型服务地址 | `"http://localhost:8000"` | 任意 HTTP 地址 |
| temperature | 生成随机性 | `0.3` | 0~1，越高结果越多样 |
| maxTokens | 最大输出长度 | `8192` | 整数 |

### 3.2 双后端模式

插件支持两种推理后端，通过 `modelBackend` 配置切换：

| 后端 | 适用场景 | 需要什么 |
|:----|:--------|:--------|
| **cloud（云端）** | 普通用户，即开即用 | 通义千问 API Key |
| **local（本地）** | 研究/高级用户，使用 LoRA 微调模型 | 自行启动 model_server.py 服务 |

**云端模式**（默认）：插件调用通义千问在线 API，响应快，无需本地计算资源。

**本地模式**：插件调用你自建的模型服务（需先启动 `model_server.py`），适合使用团队微调后的 LoRA 模型。设置方法：

```json
"hitsmartcoder.modelBackend": "local",
"hitsmartcoder.localEndpoint": "http://localhost:8000"
```

> 本地模式下不需要配置 `apiKey`，因为模型服务运行在你的本地机器上。

**修改示例**：在 settings.json 中添加：

```json
"hitsmartcoder.apiKey": "你的key",
"hitsmartcoder.completionMode": "hint",
"hitsmartcoder.modelBackend": "cloud"
```

## 4. 教学引导模式

插件提供四种交互模式，适合不同学习阶段（通过 `hitsmartcoder.completionMode` 切换）。

| 模式 | 行为 | 适合场景 |
| --- | --- | --- |
| standard | 直接给出完整代码 | 效率优先，日常开发 |
| hint | 给出解题思路和方向性提示，不写任何代码 | 算法学习，独立思考 |
| scaffold | 给出代码框架（函数签名、循环骨架），核心逻辑留空标注 TODO | 知道思路但不知道怎么写框架 |
| explain | 解释涉及的概念和原理，给出分步思路和引导性问题 | 完全不理解，想先学明白 |

#### hint 模式示例

- 用户输入函数签名 `def binary_search(arr, target):`
- AI 返回提示：

> 💡 **思路**：设定左右指针，每次取中间值与目标比较，直到找到或区间为空。
> 🔑 **关键步骤**：计算 mid，判断 arr[mid] 与 target 的大小，移动左右边界。
> （不会给出完整代码，请自行实现）

#### compare 模式示例

- 返回两个候选：
  - 候选 A：递归实现（代码 + 注释）
  - 候选 B：循环实现（代码 + 注释）
- 用户可通过键盘选择插入哪个实现。

#### 使用方式

1. 在 VS Code 设置中修改 `hitsmartcoder.completionMode` 为 desired 模式
2. 打开代码文件，光标放在函数体内
3. 按 `Ctrl+Shift+P` → 输入 `AI: 教学引导`
4. 在弹出的输入框中描述你要实现的功能
5. AI 会根据你选的教学模式给出不同深度的引导

## 5. 常见问题（FAQ）

**Q：插件提示「API Key 无效」怎么办？**
A：检查设置中的 `hitsmartcoder.apiKey` 是否正确，且通义千问账户仍有可用额度。如果使用本地模式，不需要配置 API Key。

**Q：补全结果不准确？**
A：尝试调低温度（如 0.1）获得更稳定输出，或切换到本地模式使用微调模型。

**Q：支持哪些语言？**
A：理论上支持所有 VS Code 可识别语言，已重点测试 Python、JavaScript、C++、Java、Go。

**Q：云端模式和本地模式有什么区别？**
A：云端模式即开即用，调用通义千问在线 API，需要联网和 API Key。本地模式需要自行启动模型服务（model_server.py），适合使用团队 LoRA 微调后的模型，不依赖网络。

**Q：怎么切换到本地模式？**
A：在 VS Code 设置中修改 `hitsmartcoder.modelBackend` 为 `"local"`，并确保 `model_server.py` 已在本地运行。详见第 6 节。

## 6. 高级：使用 LoRA 微调模型

### 6.1 前提条件

- Python 3.10+ 环境（需安装 torch, transformers, peft, fastapi, uvicorn）
- LoRA adapter 权重文件（位于 `output/coder_l2/`，已训练好的最优版本）
- 基座模型 Qwen2.5-Coder-1.5B-Instruct（需下载到本地）

### 6.2 启动模型服务

在终端中运行：

```bash
# CPU 环境
python finetune/model_server.py \
  --base-model "D:\path\to\Qwen2.5-Coder-1.5B-Instruct" \
  --adapter-path output/coder_l2 \
  --port 8000

# GPU 环境（去掉 --no-fp16 默认启用 fp16）
python finetune/model_server.py \
  --base-model "D:\path\to\Qwen2.5-Coder-1.5B-Instruct" \
  --adapter-path output/coder_l2 \
  --port 8000
```

看到 `Server is running at http://0.0.0.0:8000/v1` 字样说明服务启动成功。

### 6.3 配置插件连接本地服务

1. 打开 VS Code 设置 (`Ctrl+,`)
2. 搜索 `hitsmartcoder.modelBackend`，改成 `"local"`
3. 确认 `hitsmartcoder.localEndpoint` 为 `"http://localhost:8000"`（默认值，一般不需要改）
4. 重新执行代码补全，插件会调用本地微调模型而非云端 API

### 6.4 部署到服务器供团队使用

如果需要让团队成员共享微调模型，可以将 `model_server.py` 部署到一台服务器上，团队成员将 `localEndpoint` 设置为服务器地址即可。
